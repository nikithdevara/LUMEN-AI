# Implementation Plan - LUMEN AI Backend

This implementation plan details the creation of a production-ready, modular, and secure backend system using **FastAPI (Python 3.12)** and **PostgreSQL (SQLAlchemy)** for LUMEN AI.

## Project Context
The frontend is already built with Base44. The backend must expose REST APIs matching the expected endpoints, structures, and AI workflows. All API responses will return a standardized JSON format:
- **Success:** `{"status": "success", "data": { ... }}`
- **Error:** `{"status": "error", "message": "error description"}`

---

## Proposed Changes

We will create the backend codebase under the `backend/` directory in the workspace.

### 1. Database Schema (PostgreSQL via SQLAlchemy ORM)

We will define 10 tables in SQLAlchemy:
1. `roles`: Contains predefined roles (`Student`, `Parent`, `Hotel Staff`, `Volunteer`).
2. `users`: Stores user info, hashed password, and reference to `role_id`.
3. `stories`: Interactive scenarios tailored for target roles.
4. `story_scenes`: Individual scenes for each story including character information, scenario text, choices (stored as JSON), and learning objectives.
5. `user_story_progress`: Tracks completion percentage and current scene index for a user on a given story.
6. `user_choices`: Captures the decisions made by users in scenes and the corresponding AI explanations.
7. `reflections`: Holds user-written reflections, summaries generated by AI, and key lessons.
8. `quiz_questions`: Stores MCQs, option list, correct answers, and explanations.
9. `quiz_results`: Records user quiz scores and completion timestamps.
10. `resources`: Reference articles, guidelines, awareness materials, and URLs.

### 3. AI Service & Safety Layer
A unified `ai_service.py` handles requests to either **OpenAI** (via `openai`) or **Google Gemini** (via standard api/sdk calls) depending on active environment variables.

All requests pass through a structured prompt safety layer enforcing:
- Survivor-centered language and educational tone.
- Avoidance of graphic descriptions, victim blaming, and stereotypes.
- Strict compliance with JSON schema format.

---

### 4. REST API Endpoint Designs

#### Auth:
* `POST /api/auth/register`: Create user.
* `POST /api/auth/login`: Validate credentials, return JWT access token.

#### Users:
* `GET /api/users/profile`: Return current logged-in user profiles.
* `PUT /api/users/role`: Update role of the logged-in user (e.g., student, parent, hotel staff, volunteer).

#### Stories:
* `GET /api/stories`: Get stories filtered by user's role.
* `POST /api/story/start`: Load the initial scene of a story for a user.
* `POST /api/story/continue`: Advance the story based on choice selected, calculating next scene.

#### AI Interactions:
* `POST /api/ai/explain-choice`: Evaluate choices and provide safety insights.
* `POST /api/ai/reflection`: Empathic reflection breakdown (summary, key lessons, growth area).
* `POST /api/ai/generate-quiz`: Generate quiz questions dynamically if they don't exist yet for the story.
* `POST /api/quiz/submit`: Submit answers, calculate score, increment story/user progress.
* `GET /api/recommendations`: Fetch role-specific awareness resources.
